💡 Reasoning Behind Every Step

StepWhat It DoesWhy It Exists
1. Connect via WSS Opens a secure WebSocket connection to the Coinbase Advanced Trade API. WebSocket provides low-latency, push-based streaming — far more efficient than polling a REST endpoint for real-time ticker data.
2. Subscribe to Ticker Sends a JSON subscribe message for configured products (BTC-USD, ETH-USD, …). Coinbase requires explicit subscription within 5 s of connecting. Centralising the product list in config makes it easy to add/remove pairs without code changes.
3. Receive (Time-Bounded) Reads messages for exactly STREAM_DURATION_SECONDS (default 60 s). Makes every Airflow run idempotent — each execution captures a fixed window of data tagged with the logical date, so re-runs don't duplicate or miss data.
4. Schema Validation Each incoming message is validated against TICKER_RAW_SCHEMA. Prevents malformed or unexpected upstream changes from corrupting the data lake. Invalid records are logged and dropped early.
5. Metadata Enrichment Adds _ingested_at (UTC timestamp) and _execution_date (Airflow logical date). _ingested_at enables latency analysis. _execution_date supports S3 partition pruning and Redshift WHERE filters for efficient queries.
6. Type Casting Converts string prices/volumes to Decimal; normalises timestamps to UTC. Coinbase sends all numbers as strings. Decimal prevents floating-point rounding in financial calculations.
7. Analytics Enrichment Computes derived columns: spread, mid-price, 24 h change %, OHLC. Pre-computing commonly needed metrics reduces Redshift compute costs and keeps analytics layers thin.
8. Store to S3 Writes partitioned Parquet files under s3://bucket/transformed/. Parquet offers columnar compression (~10x vs JSON) and predicate push-down. Partition by date enables cheap, targeted reads.
9. Load into Redshift Uses COPY from S3 into Redshift Serverless. COPY is massively parallel and the fastest bulk-load method. Serverless auto-scales so cost tracks actual usage.

▶ How to Run

  1. Set up environment — Copy .env.example → .env and fill in AWS / Redshift credentials.
  2. Start Airflow — astro dev start
  3. Trigger the DAG — Open localhost:8080, find coinbase_ticker_etl, and toggle it on or trigger manually.
  4. Monitor — Check task logs in the Airflow UI for debugging and performance monitoring.
